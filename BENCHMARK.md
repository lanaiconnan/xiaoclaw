## 为什么 WASM 更快更小？

### 1. 编译时优化
```
传统: 源代码 → 解释/JIT → 运行
WASM: 源代码 → AOT 编译 → 优化字节码 → 运行
```

### 2. 体积来源
- **传统方案**: 运行时 + 解释器/JIT + 标准库 + 依赖
- **WASM**: 纯业务代码，无运行时

### 3. 启动优化
- **传统**: 需要解析、验证、JIT 编译
- **WASM**: 直接执行预编译字节码
